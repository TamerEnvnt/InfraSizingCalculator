UX States Review - v0.4 Wireframes

This document contains visual mockups for UX states that need review before implementation. Each state should be evaluated for clarity, accessibility, and consistency with the design system.

Error States REVIEW REQUIRED

Error states must be clear, actionable, and non-blocking where possible. Review each state for message clarity and recovery options.

API Failure Banner HIGH PRIORITY
!

Unable to calculate sizing

The calculation service is temporarily unavailable. Your configuration has been saved and will be used when the service recovers.

Design decisions:
  • Non-modal banner allows continued viewing of last-known-good results
  • Retry button gives immediate action path
  • Red accent signals error without being alarming

Loading States REVIEW REQUIRED

Loading states should provide visual feedback without jarring transitions. Skeleton screens are preferred over spinners for content-heavy areas.

Skeleton Summary Cards HIGH PRIORITY
Design decisions:
