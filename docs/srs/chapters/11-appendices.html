Appendix D: Glossary

Definitions of terms used throughout this document

TermDefinition
Blazor ServerA web framework by Microsoft that runs Razor components on the server, sending UI updates over SignalR WebSocket connection.
Cluster ModeKubernetes deployment topology: Multi-Environment (all envs in one cluster), Shared Cluster (some envs share), or Per-Environment (dedicated clusters).
DR (Disaster Recovery)Strategies for recovering from catastrophic failures, including Cold, Warm, and Hot standby patterns.
HA (High Availability)System design approach to ensure continuous operation, typically using redundant components (Active-Passive, Active-Active, N+1).
HeadroomAdditional resource capacity (expressed as percentage) reserved for workload spikes and operational overhead.
Infrastructure NodesKubernetes nodes dedicated to platform services (ingress, monitoring, logging) rather than application workloads. Required by OpenShift.
K8sAbbreviation for Kubernetes, an open-source container orchestration platform.
Low-Code PlatformDevelopment platforms that enable rapid application development with minimal hand-coding (e.g., Mendix, OutSystems).
Master NodesKubernetes control plane nodes that manage cluster state, scheduling, and API operations.
Overcommit RatioThe ratio of allocated resources to physical capacity, allowing more workloads than physical resources by assuming not all will peak simultaneously.
PAYGPay-As-You-Go pricing model where charges are based on actual usage without upfront commitment.
ReplicaA copy of an application instance running simultaneously for availability and load distribution.
Reserved InstancesCloud pricing model with discounted rates in exchange for commitment to use resources for 1-3 years.
SignalRMicrosoft library for adding real-time web functionality, used by Blazor Server for server-client communication.
SQLiteLightweight, serverless, embedded SQL database engine used for local data persistence.
VM (Virtual Machine)Traditional virtualized server deployment model as opposed to containerized Kubernetes deployment.
Worker NodesKubernetes nodes that run application workloads (pods).
