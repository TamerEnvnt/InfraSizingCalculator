Example: Complete Use Case

UC-001

Calculate Kubernetes Cluster Sizing

1. Identification
Use Case IDUC-001
Use Case NameCalculate Kubernetes Cluster Sizing
Version1.0
StatusApproved
2. Overview
Brief Description Allows infrastructure architects to calculate the required Kubernetes cluster sizing (nodes, CPU, RAM, storage) based on application requirements, technology stack, and deployment configuration.
Priority P1 (Critical) - Core functionality
3. Actors
Primary Actor Infrastructure Architect
Description: Technical professional responsible for designing infrastructure solutions
Skills: Expert-level understanding of Kubernetes and cloud infrastructure
7. Business Rules
Rule IDRule DescriptionImplementation
BR-M001 Managed Kubernetes services (EKS, AKS, GKE, OKE) have 0 master nodes If distribution.HasManagedControlPlane == true, set masterNodes = 0
BR-M002 Large clusters (100+ workers) require 5 control plane nodes If workerNodes >= 100, set masterNodes = 5
BR-W001 Minimum 3 worker nodes per cluster for HA workerNodes = Max(calculatedNodes, 3)
BR-W002 15% of node resources reserved for system overhead availableResources = nodeResources × 0.85
BR-H001 Production environments require 37.5% headroom prodNodes = baseNodes × 1.375
8. Main Success Scenario
1
Actor: Selects Kubernetes distribution from available options
UI Element: Distribution selection cards (PlatformStep.razor)
Options: OpenShift, EKS, AKS, GKE, Rancher, K3s, etc. (38+ options)
Applies: BR-M001 (managed vs self-hosted detection)
2
System: Loads distribution-specific configuration and displays technology options
Service: DistributionService.GetDistributionConfig()
Data Loaded: Node specs, features (HasInfraNodes, HasManagedControlPlane)
3
Actor: Selects application technology (.NET, Java, Node.js, etc.)
UI Element: Technology selection cards (TechnologyStep.razor)
Display: Shows resource requirements per tier (S/M/L/XL)
4
Actor: Configures environments and enters application counts
Environments: Dev, Test, Stage, Prod, DR (checkboxes)
App Counts: Per-tier counts (Small, Medium, Large, XLarge)
Validation: See VR-001, VR-002
5
System: Validates all input data
Validation: Range checks, required fields, cross-field rules
On Failure: Go to Exception Flow E2
6
Actor: Clicks "Calculate Sizing" button
UI Element: Primary button in WizardNavigation.razor
7
System: Executes sizing calculation
Service: K8sSizingService.CalculateSizing()
Applies: BR-M001 BR-M002 BR-W001 BR-W002 BR-H001
Performance: Must complete within 500ms
8
System: Displays sizing results
Display: ResultsTable showing per-environment and grand totals
Includes: Master nodes, Infrastructure nodes (if OpenShift), Worker nodes
Format: CPU cores, RAM (GB), Disk (GB)
