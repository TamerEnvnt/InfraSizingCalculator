UC-013

Configure Server Roles

Overview
Brief DescriptionAllows the user to configure VM server roles specific to the selected technology. Each technology has different required roles (e.g., Web Server, App Server, Database, Controller for OutSystems).
PriorityP1 (Critical)
ComplexityHigh
Applicable Business Rules
RulesBR-VM001 BR-VM002 BR-VM003 BR-VM004
Main Success Scenario
1
System: Loads available roles for selected technology
Example roles for OutSystems:
- Controller (max: 1)
- Frontend Server (min: 2 for HA)
- Database Server
- Cache Server
2
Actor: Configures instance count per role
3
System: Validates against min/max constraints
4
Actor: Optionally adjusts resource specs per role
Test Scenarios

UC-014

Configure HA Pattern

Overview
Brief DescriptionAllows the user to select a High Availability pattern for VM deployments. HA patterns determine how VMs are replicated for fault tolerance.
PriorityP1 (Critical)
ComplexityMedium
Applicable Business Rules
RulesBR-VM005
Main Success Scenario
1
System: Displays HA pattern options
Options with multipliers:
- None (1.0x)
- Active-Active (2.0x)
- Active-Passive (2.0x)
- N+1 (1.5x)
- N+2 (1.67x)
2
Actor: Selects HA pattern
3
System: Updates VM count calculations
Test Scenarios
IDDescriptionExpected ResultPriority
TS-014-01Select Active-Active HAVM count doubledP1
TS-014-02Select N+1 HAVM count x 1.5P1
TS-014-03Select NoneBase VM count (no redundancy)P1
Traceability
Related RequirementsREQ-FR-014
ComponentsHAPatternSelector.razor, VMSizingService.cs
UC-015

Configure DR Pattern

Overview
Brief DescriptionAllows the user to configure Disaster Recovery patterns for VM deployments. DR patterns determine how infrastructure is replicated across sites for business continuity.
PriorityP2 (High)
ComplexityMedium
Main Success Scenario
1
System: Displays DR pattern options
Options:
- None (no DR site)
- Pilot Light (minimal DR footprint)
- Warm Standby (scaled down DR)
- Hot Standby (full replica)
2
Actor: Selects DR pattern
3
System: Adds DR site VMs to calculations
Test Scenarios
IDDescriptionExpected ResultPriority
TS-015-01Select Hot StandbyFull DR environment addedP1
TS-015-02Select Pilot LightMinimal DR VMs addedP2
Traceability
Related RequirementsREQ-FR-015
ComponentsDRPatternSelector.razor, VMSizingService.cs
