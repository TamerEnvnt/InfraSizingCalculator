4.1 Wizard Steps (UC-001 to UC-005)

Step-by-step configuration wizard for platform, deployment model, and technology selection

UC-001

Select Platform Type

Overview
Brief DescriptionAllows the user to select whether they are sizing for native applications (.NET, Java, Node.js, etc.) or low-code platforms (Mendix, OutSystems). This is the first step in the configuration wizard.
PriorityP1 (Critical) - Entry point to application
ComplexityLow
Actors
Primary ActorInfrastructure Architect - Technical professional starting the sizing process
Secondary ActorsDevOps Engineer, Platform Engineer
Preconditions
#Precondition
PRE-1User has accessed the application via web browser
PRE-2Application has fully loaded (JavaScript initialized)
Postconditions
#Postcondition
POST-1Platform type is stored in application state
POST-2Available technologies are filtered based on platform selection
POST-3User is advanced to Deployment Step (UC-002)
Main Success Scenario
1
System: Displays platform selection cards
UI Element: PlatformStep.razor
Options Displayed:
- Native Applications (icon: code brackets) - ".NET, Java, Node.js, Python, Go"
- Low-Code Platforms (icon: blocks) - "Mendix, OutSystems"
2
Actor: Clicks on desired platform card
Interaction: Single click to select
Visual Feedback: Selected card shows highlighted border and checkmark
3
System: Updates application state with selection
State Update: ConfigState.PlatformType = selected value
Action: Filters available technologies for next step
4
Actor: Clicks "Next" button or card is auto-advancing
5
System: Advances to Deployment Step (UC-002)
Alternative Flows
A1: Change Selection (from Step 2)

Condition: User clicks different platform card after initial selection

A1.1
System: Updates selection to new platform
A1.2
System: Clears any dependent selections (technology, distribution)

Rejoins Main Flow: Step 3

Exception Flows
E1: No Selection Made

Trigger: User clicks "Next" without selecting a platform

E1.1
System: Displays inline validation message "Please select a platform type"
E1.2
System: Keeps user on current step

Recovery: User selects a platform (returns to Step 2)

Input Data Requirements
FieldTypeRequiredConstraintsDefault
PlatformTypeEnumYesNative, LowCodeNone (must select)

UC-002

Select Deployment Model

Overview
Brief DescriptionAllows the user to choose between Kubernetes container orchestration or traditional Virtual Machine deployment. This fundamental choice determines which configuration flow to follow.
PriorityP1 (Critical) - Determines entire workflow
ComplexityLow
Main Success Scenario
1
System: Displays deployment model selection cards
Options:
- Kubernetes (icon: wheel/helm) - "Container orchestration with pods and clusters"
- Virtual Machines (icon: server) - "Traditional VM-based deployment"
2
Actor: Clicks on desired deployment model card
3
System: Updates application state
Side Effects:
- K8s: Enables K8s configuration panels, disables VM panels
- VM: Enables VM configuration panels, disables K8s panels
4
Actor: Clicks "Next" button
5
System: Advances to Technology Selection Step

UC-005

Select Cluster Mode

Overview
Brief DescriptionAllows the user to choose between Multi-Cluster (separate cluster per environment), Shared Cluster (single cluster for all environments), or Per-Environment (calculate for a single selected environment). This determines how environment resources are aggregated.
PriorityP1 (Critical) - Determines calculation strategy
Main Success Scenario
1
System: Displays cluster mode options
Options:
- Multi-Cluster: "Separate cluster per environment (Dev, Test, Stage, Prod, DR)"
- Shared Cluster: "Single cluster with namespace isolation for all environments"
- Per-Environment: "Calculate sizing for a single environment only"
2
Actor: Selects cluster mode
3
System: Updates state and adjusts UI
Multi-Cluster: Shows all environment configuration panels independently
Shared Cluster: Shows all environments but calculates unified totals
Per-Environment: Shows single environment selector dropdown
4
Actor: Clicks "Next" button
5
System: Completes wizard, transitions to main configuration view
Input Data Requirements

SelectedEnvironmentEnumConditionalRequired if PerEnvironment modeProd
