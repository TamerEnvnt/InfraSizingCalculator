Legend & Sizing Guidelines

Environment Types

Development - Minimal specs, single instance
Testing - Dev-like specs, integration testing
Staging - Production-like, user acceptance
Production - Full specs, HA enabled
DR - Standby for disaster recovery

HA Multipliers

Dev/Test: 1x (no HA)
Stage: 2x (Active-Passive)
Prod: N+1 (Clustered)
DR: Matches Prod (standby)

MaxInstances Constraints

OutSystems Controller: MaxInstances = 1
OutSystems LifeTime: MaxInstances = 1
OutSystems Frontend: Unlimited (scalable)
OutSystems Database: Clusterable
